This is my digital clock I made through Action Script 3.0. It consists of two class
files, one that draws the digits and one that updates the digits according to a 'ticker'
that receives the time from the computer.

Finding out how to draw the digits onto the clock was very straightforward, the problem
was finding out how to remove them and redraw them so they don't all become 8's
(because an 8 digit has every segment in the digit drawn). This was solved by
utilizing the function this.graphics.clear() in the correct position to remove
the digit before drawing another one.

After that was solved, the next problem was finding out how to convert a two
digit numbers like "12' into two one digit numbers so that the digits can be
drawn. This was solved by using modulo (%) to divide the numbers by 10 and
give the remainder. For the example 12, 12 % 10 would give us 2. This is
would provide us with the necessary digit to draw the last number of our
time.

I actually did something really stupid with drawing the digits, as I did the
above for the LAST digits of my time, but NOT my FIRST digits. I came up with
the absolutely ridiculous solution of defining every 10 digit interval for
what digit to draw for the 10's place. An example is to check if the hour
is less that 20 and greater than 9, if this is true, draw a "1" in the 10's
place. If the hour is less than 25 and greater then 19, draw a "2" in the
10's place.

This carried on for MANY lines of code...

This is my digital clock with the added functionality of displaying milliseconds.

It was a relatively easy addition, as the code to draw and receive the current
milliseconds was almost the exact same as the rest of the digits.

A difference that did come up was that if you trace() or print the milliseconds,
you might get a number with three digits instead of two because milliseconds are
a thousandths of a second, not a hundredth!

To follow up with that, my stupid solution was to define every 100 second interval
instead of just using % to find the first number of the milliseconds.

I am regret.

The main problem was the fact that the milliseconds couldn't update
fast enough because a thousandths of a second is pretty fast! To fix this, I raised
the frame rate until a milliseconds looked like they were going "fast enough".

The next addition to my digital clock was 'shadows' in the background. The purpose
of this is to imply that there are segments that are not being used to display the
current time. It is a purely aesthetic addition, but probably more useful for actual
physical digital clocks.

It was quite simple, as all I did was make a separate class to draw the shadows.
It basically drew a darkened eight digit using the exact same drawing commands as
the other digits.

The main problem that came up was the shadows were drawn on top of the digits!
This just made every digit display a darkened 8. The quick and easy fix was to
draw the shadows first, and then have the actual digits drawn on top of the shadows.
This was done by simply rearranging the order of the code.

Someone who is really good at observing things may notice that the milliseconds
of this clock eventually start to not match with the milliseconds of the above
clock. I believe this is because the computer really can't update the digits fast
enough at the EXACT same time, so we eventually see nuances.
